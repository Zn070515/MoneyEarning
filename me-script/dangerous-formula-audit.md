# 危险公式审计

> 版本 v1.0 | 2026-05-24 | 识别可能使 ME Script 引擎崩溃、死循环、或产生误导结果的输入

---

## 一、崩溃类风险

### 1.1 除零

**风险公式**：
```
X:=C/(HHV(H,10)-LLV(L,10));
```
当 10 日内最高=最低（一字板），分母=0 → NaN/Inf

**当前防护**：运行时 panic 或返回 NaN
**建议**：编译时静态分析检测除法操作，警告用户添加保护条件

### 1.2 空数据引用

**风险公式**：
```
X:=MA(C,200);
CROSS(MA5, MA10)
```
如果数据 <200 根 → MA(200) 返回 NaN → 后续计算全部 NaN

**当前防护**：部分函数有 `period == 0` 检查（已修复），但无通用数据量检查
**建议**：在回测入口检查数据量，不足时拒绝执行并给出友好提示

### 1.3 超大数组

**风险公式**：
```
SUM(C, 999999)
```
参数超大 → 内存分配过大 → WASM 内存溢出

**当前防护**：无上限检查
**建议**：编译时对参数值做范围检查（n < 10000）

### 1.4 索引越界

**风险公式**：
```
REF(C, 99999)
```
偏移量超出数据范围 → 返回 NaN（假设有防护）或 panic

**当前防护**：见 runtime.rs
**建议**：执行时检查偏移量，超范围返回 NaN 并记录告警

## 二、性能黑洞类

### 2.1 嵌套深度

**风险公式**：
```
HHV(HHV(HHV(HHV(HHV(C,5),5),5),5),5)
```
嵌套 5+ 层 → 指数级计算量

**建议**：编译时限制 AST 深度（max_depth = 50）

### 2.2 超大循环展开（隐含）

ME Script 虽然无显式循环，但 Series 操作隐含沿时间轴的全序列计算。
超大序列（如 10 万根K线）× 多层嵌套 = 潜在 O(n²) 复杂度。

**建议**：限制每次执行的最大数据点数（如 50,000 bar）

## 三、恶意输入类

### 3.1 超长脚本

**风险**：粘贴超长文本（>10MB）
**建议**：限制脚本长度 ≤ 100,000 字符

### 3.2 递归函数调用

**风险**：通过自我引用制造隐式递归
**当前状态**：ME Script 不支持函数定义，暂不适用。未来如添加自定义函数需考虑。

### 3.3 Unicode 混淆

**风险**：使用相似 Unicode 字符混淆变量名（如用西里尔字母 А 替代拉丁 A）
**建议**：标识符仅允许 ASCII 字母+数字+下划线+中文

## 四、已在 v0.11.0 修复的已知问题

| 问题 | 文件 | 修复方式 |
|------|------|---------|
| period=0 除零 | atr.rs, adx.rs, supertrend.rs | 入口检查 `period == 0` |
| ad_line unwrap | adosc.rs | 改为 `Result` 返回 |
| rvgi unwrap | inertia.rs | 改为 `Result` 返回 |
| 排序 unwrap | median.rs | `unwrap_or(Ordering::Equal)` |
| pool_size=0 | search.rs | 入口检查 `pool_size == 0` |
| 协方差 n<2 | stats.rs | 入口检查 `n < 2` |
| 特征值 n==0 | stats.rs | 入口检查 `n == 0` |
