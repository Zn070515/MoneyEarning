# 公式错误提示指南

> 版本 v1.0 | 2026-05-24 | ME Script 错误提示规范

---

## 一、设计原则

错误提示必须让**不懂编程的通达信用户**也能看懂：
1. 告诉用户**哪一行错了**（行号）
2. 告诉用户**为什么错了**（中文描述，不用技术术语）
3. 告诉用户**怎么改**（提示或替代写法）
4. 绝不显示 `panic`、`undefined`、`NaN`、`Rust backtrace`

## 二、错误类型与标准提示

### 2.1 语法错误

```
❌ 第5行 第12列：语法错误
   SMA(CLOSE,5,1)
          ^
   此处不期望逗号。SMA 函数需要3个参数：SMA(序列, 周期, 权重)。
   示例：SMA(CLOSE, 5, 1)
```

### 2.2 不支持的函数

```
❌ 第3行：不支持的函数 "DRAWLINE"
   DRAWLINE(条件1, 价格1, 条件2, 价格2, 0)
   ^^^^^^^^
   当前版本不支持 DRAWLINE（条件画线）函数。
   替代方案：使用 plot 配合条件表达式绘制信号点。
   示例：plot 突破信号 = if(突破条件, high, null) { style: plot.circle };
```

### 2.3 参数数量错误

```
❌ 第3行：函数 "MA" 参数数量错误
   MA(CLOSE)
   ^^
   MA 函数需要2个参数：MA(序列, 周期)
   不需要的括号可能表示错误的函数调用。
   示例：MA(CLOSE, 5) 或 MA(CLOSE, 20)
```

### 2.4 变量未定义

```
❌ 第4行：未定义的变量 "MA5"
   CROSS(MA5, MA10)
         ^^^
   你在使用 MA5 之前没有定义它。
   是否想写：
     let ma5 = MA(CLOSE, 5);
     CROSS(ma5, MA10)
```

### 2.5 除零警告

```
⚠️ 第6行：可能除以零
   (C - LLV(L, 9)) / (HHV(H, 9) - LLV(L, 9))
                      ^^^^^^^^^^^^^^^^^^^^^^^^
   当 HHV(H,9) 等于 LLV(L,9) 时（一字板或横盘），
   该表达式会导致除零错误。
   建议加保护：
     let range = HHV(H,9) - LLV(L,9);
     let rsv = if(range > 0, (C - LLV(L,9)) / range * 100, 50);
```

### 2.6 数据类型不匹配

```
❌ 第7行：条件表达式类型不匹配
   IF(C > O, 1, "买入")
                  ^^^^
   IF 函数的真假分支必须返回相同的类型。
   此处：真分支返回 数字(1)，假分支返回 字符串("买入")
   建议：统一为数字类型，如 IF(C>O, 1, 0)
```

### 2.7 前向引用检测

```
❌ 第8行：禁止使用前向引用
   REF(C, -1)
        ^^^^
   前向引用（负数偏移）会使用未来数据，在交易信号中不允许。
   这会让回测结果不可信。
   如果你需要引用昨天的数据，请使用 REF(C, 1)（正数=向过去偏移）。
```

## 三、运行时错误提示

### 3.1 数据不足

```
⚠️ 回测结果不可用：数据不足

   该股票只有 30 根日K线，而策略"双均线交叉"
   至少需要 60 根K线才能计算出有效信号。

   建议：
   · 选择数据量更多的股票（至少252根=约1年）
   · 或使用更短周期的参数（如 MA5 替代 MA20）
```

### 3.2 策略无交易

```
ℹ️ 无交易信号

   策略"RSI极值反转"在所选数据区间内未产生任何交易信号。
   可能原因：
   · RSI 未触及设定的极端值（<30 或 >70）
   · 数据区间内趋势性太强（RSI 长期在一侧运行）
   · 参数设置过于严格

   建议：检查参数设置或更换回测区间
```

### 3.3 编译超时

```
⚠️ 脚本编译超时

   脚本可能包含过于复杂的计算或死循环。
   请检查是否存在：
   · 嵌套过深的 IF 表达式
   · 在不需要的地方使用了循环逻辑
   · 超大参数范围

   建议：简化脚本结构或联系支持
```
