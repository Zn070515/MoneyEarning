# 杀毒误报处理计划

> 版本 v1.0 | 2026-05-24

---

## 一、为什么会误报

未签名的 .exe/.msi 文件在以下场景最易被误报：
1. 文件较新（发布<1周），杀软厂商尚未收录
2. 文件包含打包的 Rust 二进制（与恶意软件共用某些系统调用模式）
3. 安装器使用了压缩/加壳（Tauri NSIS 安装器天然压缩）
4. 没有 EV Code Signing 数字签名

## 四、误报提交流程

### Windows Defender（免费，建议优先提交）

1. 访问 https://www.microsoft.com/en-us/wdsi/filesubmission
2. 提交文件 + 说明 "This is a legitimate desktop application, false positive"
3. 通常 24-48 小时内处理

### 360安全卫士

1. 访问 https://open.soft.360.cn/report.php
2. 提交软件信息和安装包
3. 需提供营业执照或身份证明（个人开发者可尝试）

### 腾讯电脑管家

1. 访问 https://guanjia.qq.com/user_complaint.html
2. 提交误报申诉

### VirusTotal 监控

1. 每次发版后上传到 VirusTotal
2. 记录误报引擎数量
3. 如果 >5 个引擎误报 → 向对应厂商分别提交
4. 链接保存到发版文档

## 五、长期解决方案

| 方案 | 成本 | 时间 | 效果 |
|------|:--:|:--:|------|
| 购买 EV Code Signing 证书 | ¥2000-3500/年 | 1-2周 | 彻底解决 SmartScreen + 大幅减少误报 |
| 每年向各杀软提交白名单 | 免费但耗时 | 每次发版1-2天 | 逐步累积信誉 |
| 开源部分代码 + 公布 SHA256 | 免费 | 即时 | 增加可信度，但不影响杀软判断 |

- [ ] VirusTotal 扫描 .exe 和 .msi
- [ ] 记录误报引擎数量和名称
- [ ] 向误报厂商提交 false positive 报告
- [ ] 在下载页更新 SHA256 校验值
- [ ] 更新本文档的误报记录
