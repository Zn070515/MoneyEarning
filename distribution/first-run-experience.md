# 首次使用体验设计

> 版本 v1.0 | 2026-05-24 | 目标：用户打开后 3 分钟内看到第一张 K 线图

---

## 一、设计目标

| 时间 | 用户行为 | 目标 |
|:----:|---------|------|
| 0-30秒 | 打开应用，看到欢迎页 | 不面对空白界面 |
| 30秒-1分钟 | 选择"加载演示数据" | 不需要手动导入任何文件 |
| 1-2分钟 | 自动打开第一只演示股票 K 线 | 看到蜡烛图 + 成交量 |
| 2-3分钟 | 添加一个技术指标（如 MACD） | 理解"可以叠加指标" |
| 3-5分钟 | 跑一次示例回测 | 理解核心价值：回测验证想法 |

---

## 二、首次启动流程

### 步骤 0：检测首次启动

```
App 启动 → 检查 localStorage("has_completed_onboarding")
  ├─ false → 显示首次使用向导
  └─ true  → 正常进入 Dashboard
```

### 步骤 1：欢迎页

```
┌─────────────────────────────────────────────┐
│                                             │
│          QuantVault                         │
│          本地离线行情数据分析工具            │
│                                             │
│   ⚠️ 重要提示                               │
│   本软件不提供任何投资建议。所有功能仅供      │
│   历史数据分析与自我研究。投资有风险，        │
│   入市需谨慎。                               │
│                                             │
│   [我已了解，开始使用]                       │
│   [查看完整风险提示]                         │
│                                             │
└─────────────────────────────────────────────┘
```

用户必须点击"我已了解"才能继续。——这确保每个用户都至少看过一次风险提示。

### 步骤 2：选择上手方式

```
┌─────────────────────────────────────────────┐
│   选择入门方式                               │
│                                             │
│   ┌─────────────┐ ┌─────────────┐ ┌───────┐│
│   │ 📊           │ │ 📁           │ │ 📖    ││
│   │ 加载演示数据  │ │ 导入我的数据  │ │看教程  ││
│   │             │ │             │ │       ││
│   │ 一键生成10只  │ │ 导入通达信   │ │ 功能介绍││
│   │ 演示股票数据  │ │ .day 文件    │ │ 视频图文││
│   │ 即刻上手体验  │ │ 使用已有数据  │ │ 快速了解││
│   └─────────────┘ └─────────────┘ └───────┘│
│                                             │
│   [跳过，直接进入软件]                       │
└─────────────────────────────────────────────┘
```

### 步骤 3a：加载演示数据（推荐路径）

```
点击"加载演示数据"
  → invoke("seed_demo_data")  // 已有函数，幂等
  → 显示加载动画："正在生成演示数据..."
  → 完成后自动跳转：打开 000001.DEMO K线图
  → K 线图自动加载完毕，用户看到蜡烛图 + 成交量柱
  → 右下角 toast："演示数据已加载，10只演示股票可查看"
  → 进入正常应用界面
```

### 步骤 3b：导入数据（高级路径）

```
点击"导入我的数据"
  → 跳转到数据导入页面
  → 显示导入教程（CSV / 通达信 .day）
  → 引导用户选择文件
  → 导入完成后自动打开第一只导入的股票
```

### 步骤 3c：看教程

```
点击"看教程"
  → 展开功能介绍卡片列表
  → 每张卡片：功能图标 + 一句话说明 + "试试看"按钮
  → 点击"试试看"跳转到对应功能页
```

## 三、首次使用后的引导

### 加载演示数据成功后

- 顶部显示引导条 `💡 试试添加一个技术指标 — 点击左侧"指标" → 选择 MACD → 叠加到图表`
- 引导条可关闭，关闭后不再出现
- 完成 3 个操作后引导条自动消失：
  1. 看到 K 线图 ✅
  2. 添加了一个指标 ✅
  3. 切换了股票 ✅

### 3天后检查

```
如果 3 天后：
- 用户尚未跑过一次回测 → Dashboard 显示 "💡 还没试过策略回测？"
- 用户尚未创建预警 → Dashboard 显示 "💡 设置预警，条件触发时提醒你"
```

## 四、空状态处理

| 场景 | 空状态文案 + 操作 |
|------|-----------------|
| 数据库无股票 | "暂无股票数据。请加载演示数据或导入数据。" [加载演示] [导入数据] |
| 无自选股 | "还没有自选股。点击搜索栏添加关注的股票。" |
| 图表无数据 | "请从左侧选择一只股票加载K线数据。" |
| 回测无结果 | "尚未运行回测。选择策略模板，设置参数后点击'运行回测'。" |
| 预警无规则 | "暂无预警规则。点击'新建预警'创建你的第一条预警条件。" |
| 扫描无结果 | "未找到完全匹配的股票。尝试放宽筛选条件或减少条件数量。" |
| 组合无持仓 | "暂无持仓记录。请在'交易日志'中记录交易以自动生成持仓。" |

## 五、错误状态处理

| 场景 | 错误状态显示 |
|------|------------|
| K线引擎初始化失败 | "图表引擎初始化失败，请刷新页面重试。" [刷新] |
| 数据下载失败 | "网络请求失败，请检查网络连接后重试。" [重试] |
| 文件导入格式错误 | "文件格式不正确。请确认选择了有效的 CSV 或通达信 .day 文件。" |
| 回测失败 | "回测执行失败：[错误信息]。请检查策略参数或数据完整性。" |
| 数据库错误 | "本地数据库异常。请尝试重启应用，如问题持续请备份后恢复数据库。" |
