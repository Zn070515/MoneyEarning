## 二、L2 中风险功能审计

### 2.1 策略回测（基本）

- **文件**: `crates/wasm-backtest/src/lib.rs`, `packages/ui/src/BacktestPanel.tsx`, `packages/app/src/pages/BacktestPage.tsx`
- **风险**: 用户可能将回测收益视为预期收益
- **当前状态**: 回测结果直接展示年化收益、Sharpe、最大回撤，**无任何风险提示**
- **修改方案**:
  1. BacktestPanel.tsx 结果区顶部加 `⚠️ 历史回测不代表未来收益。策略参数可能过拟合。` 横幅
  2. BacktestPage.tsx 同理
  3. exportReport.ts 报告加头尾风险声明
- **状态**: ❌ 未修改

### 2.2 Walk-Forward 分析

- **文件**: `crates/wasm-backtest/src/walk_forward.rs`
- **风险**: 用户可能认为 Walk-Forward 消除了一切过拟合风险
- **修改方案**: 结果区加 "Walk-Forward 分析可减少过拟合风险但不能完全消除"
- **状态**: ❌ 未修改

### 2.3 Monte Carlo 模拟

- **文件**: `crates/wasm-backtest/src/monte_carlo.rs`
- **风险**: 用户可能认为 Monte Carlo 结果是"科学确定"的未来收益范围
- **修改方案**: 结果区加 "Monte Carlo 模拟基于统计假设（正态分布/随机抽样），不构成收益预测"
- **状态**: ❌ 未修改

### 2.4 参数优化

- **文件**: `crates/wasm-backtest/src/optimizer.rs`
- **风险**: 网格搜索/遗传算法易过拟合，用户可能选"最优参数"直接实盘
- **修改方案**: 优化结果区加 "优化出的'最优参数'可能过拟合历史数据，实盘表现可能显著偏离"
- **状态**: ❌ 未修改

### 2.5 自定义策略回测（ME Script）

- **文件**: `crates/wasm-custom/`, `packages/app/src/pages/MEScriptPage.tsx`
- **风险**: 用户编写策略后直接实盘
- **修改方案**: 编译/执行结果区加风险提示
- **状态**: ❌ 未修改

### 2.6 回测报告导出

- **文件**: `packages/app/src/utils/exportReport.ts`
- **风险**: 报告被用户分享/晒出时无合规声明
- **修改方案**: 报告模板加头部和尾部风险声明（参考 risk-disclaimer.md 模板）
- **当前状态**: 第105行仅有简单一句 "过往业绩不代表未来表现，投资有风险，入市需谨慎"
- **状态**: ⚠️ 有基本声明，需加强

### 2.7 组合风险分析

- **文件**: `packages/app/src/pages/PortfolioPage.tsx`, `packages/ui/src/RiskPanel.tsx`
- **风险**: Sharpe/CVaR 等指标被理解为未来风险预测
- **修改方案**: 面板顶部加 "分析基于历史数据，不构成未来风险预测"
- **状态**: ❌ 未修改

### 2.8 VaR/CVaR 计算

- **文件**: `packages/app/src/pages/PortfolioPage.tsx`
- **风险**: VaR 值被理解为"最多亏多少"的保证
- **修改方案**: VaR 展示区加 "VaR 是统计估计值，不代表最大亏损上限"
- **状态**: ❌ 未修改

### 2.9 相关性矩阵

- **文件**: `packages/app/src/pages/PortfolioPage.tsx`
- **风险**: 低 — 客观统计量
- **修改方案**: 无需
- **状态**: ✅

### 2.10 筹码分布分析

- **文件**: `packages/ui/src/DistributionPanel.tsx`
- **风险**: "筹码密集区""支撑位""压力位" 可能被理解为买卖参考
- **修改方案**: 分析区加 "筹码分布为历史成交统计，不构成支撑/压力位的确定性判断"
- **状态**: ❌ 未修改

### 2.11 形态识别

- **文件**: `packages/ui/src/PatternPanel.tsx`
- **风险**: 识别出"晨星""三白兵"等看涨形态，可能被理解为买入建议

- **状态**: ❌ 未修改

## 三、L3 高风险功能审计

### 3.1 股票扫描器

- **文件**: `packages/ui/src/ScannerPanel.tsx`
- **风险程度**: 🔴 高
- **风险原因**:
  1. 最易被用户理解为"推荐股票"的功能
  2. 输出的是股票列表 + 评分/排名 → 天然带有"推荐排序"语义
  3. 如果默认排序是"评分从高到低"，等同于告诉用户"上面的更好"
- **修改方案**:
  1. 功能入口名称：确认已改为"条件扫描"（当前 UI 文本待检查）
  2. 首次进入弹窗强制免责确认
  3. 结果列表顶部固定风险提示条
  4. 默认排序列的列名改为"技术条件匹配度"（不叫"评分""推荐度""牛股指数"）
  5. 结果列表每行不得包含"推荐""建议买入""强烈看涨"等标注
- **状态**: ❌ 未修改（需全局搜索 + 批量改名）

### 3.2 盘中预警

- **文件**: `packages/app/src-tauri/src/alerts.rs`, `packages/app/src/pages/ReviewPage.tsx`
- **风险程度**: 🟡 中高
- **风险原因**:
  1. 触发通知"价格突破XX" / "均线金叉" 可能被理解为"可以买入了"
  2. 通知消息本身不带风险提示
- **修改方案**:
  1. 预警管理页面顶部加免责提示
  2. 创建预警规则时加确认提示："预警仅为条件触发通知，不构成买卖建议"
  3. 预警通知文本中加入免责简写："(仅供研究，不构成建议)"
  4. 不得预置任何"推荐预警"规则
- **状态**: ❌ 未修改

### 3.3 策略模板列表

- **文件**: `packages/ui/src/StrategyPanel.tsx`, `crates/wasm-backtest/src/strategies.rs`
- **风险程度**: 🟡 中高
- **风险原因**:
  1. 模板名称如"双均线交叉"虽然中性，但列表天然是"选一个来用" → 用户可能理解为"这些都是能赚钱的策略"
  2. 每个模板展示的回测绩效数据（年化、Sharpe）可能被理解为推荐理由
- **修改方案**:
  1. 模板选择列表顶部加"以下策略仅供研究参考，历史绩效不代表未来表现"
  2. 检查所有 20 个模板名称，确保无"高收益""稳赚"等表述
  3. 每个模板的描述中必须包含风险提示句
  4. 模板列表不得按 Sharpe 从高到低默认排列（避免暗示排序=d推荐排序）
- **状态**: ❌ 未修改

### 3.4 内置指标选股（CAPS/CGPC/MARS）

- **文件**: `crates/wasm-scanner/`, `packages/ui/src/ScannerPanel.tsx`
- **风险程度**: 🔴 高
- **风险原因**: 与扫描器类似，但算法名（"综合自适应配对扫描"等）可能制造"高科技=高确定性"的误导
- **修改方案**: 与 3.1 扫描器方案相同 + 算法名称旁加简短说明"（基于历史统计的筛选算法）"
- **状态**: ❌ 未修改

### 3.5 Alpha158 因子打分

- **文件**: `crates/wasm-factors/`
- **风险程度**: 🟡 中高
- **风险原因**: "因子打分"听起来专业，用户可能认为"打高分的=应该买"
- **修改方案**: 同扫描器 + 加 "因子有效性随市场环境变化，高分不代表未来表现"
- **状态**: ❌ 未修改

## 四、证监会法规关键条款映射

| 法规条款 | 核心要求 | QuantVault 合规映射 |
|---------|---------|-------------------|
| 《证券法》第160条 | 证券投资咨询需经核准 | QuantVault 不是投顾机构，不提供咨询服务 |
| 《暂行规定》第2条 | 投顾服务定义：提供投资分析/预测/建议 | 产品定位为"数据分析工具"而非"投资建议" |
| 《暂行规定》第16条 | 禁止承诺收益 | 所有文案禁止"稳赚""年化XX%"等承诺 |
| 《暂行规定》第19条 | 禁止代客决策 | 不提供自动交易/跟单功能 |
| 《广告法》第25条 | 招商/投资广告须有风险提示 | 所有对外的功能展示都需配风险提示 |
