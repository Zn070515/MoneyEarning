# 功能风险分级文档

> 版本 v1.0 | 2026-05-24 | 覆盖所有已实现功能

对 QuantVault 全部功能按合规风险进行四级分类，定义每级所需的 UI 防护措施。

---

## 风险等级定义

| 等级 | 名称 | 定义 | 默认 UI 措施 |
|:----:|------|------|------------|
| **L1** | 低风险 | 纯数据展示，不涉及信号/筛选/建议 | 无需额外措施 |
| **L2** | 中风险 | 历史统计/回测结果展示 | 结果区须有"历史数据不代表未来"提示 |
| **L3** | 高风险 | 涉及条件筛选/预警/策略信号 | 入口强制免责提示 + 结果区风险声明 |
| **L4** | 禁止 | 个股推荐/目标价/承诺收益/代客决策 | 不允许实现或必须完全移除 |

### L4 — 禁止（绝不可实现）

| 功能描述 | 原因 |
|---------|------|
| 具体股票推荐（"今日推荐：平安银行"） | 构成荐股 |
| 目标价预测（"目标价 15.00"） | 无法预测价格 |
| 承诺收益（"年化收益 30%"） | 无法保证，属于虚假宣传 |
| 代客自动交易/下单 | 需持牌，且数据不上传无法实现 |
| AI 选股推荐 | 同上，构成荐股 |
| 实时买卖点弹窗（"现在买入！"） | 构成即时买卖建议 |
| 盈利能力评估（"这只股票能赚 20%"） | 承诺收益 |

## 二、L3 功能 UI 措施标准

每个 L3 功能必须满足以下全部条件：

1. **入口免责**：首次进入该功能时展示免责弹窗，用户确认后方可使用
2. **结果区声明**：结果列表/图表上方固定显示风险提示条（不可关闭/不可滚动消失）
3. **命名合规**：按钮/标签/标题使用合规名称，禁用 L4 词汇
4. **无推荐排序**：扫描结果的默认排序不得使用暗示"更好"的标签（如"推荐""牛股""最佳"）。排序依据应为可解释的技术指标名称

---

## 三、UI 检查要点

- [ ] 回测结果页顶部是否有 "历史回测不代表未来收益" 提示？
- [ ] 扫描器入口是否有免责确认弹窗？
- [ ] 扫描结果列表默认排序字段名称是否合规？
- [ ] 预警管理入口是否有免责提示？
- [ ] 策略模板列表中每个模板名称是否合规？
- [ ] 回测导出的 Markdown 报告是否包含头尾风险声明？
- [ ] Dashboard/设置页是否有"风险提示"入口？
- [ ] 所有页面是否没有任何 L4 禁止表述？
