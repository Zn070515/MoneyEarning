# NEXT_STAGE_SPEC.md

# v0.12-v0.15 可卖 Alpha 闸门阶段任务书

当前状态：
项目已经完成至 v0.11.0，已有专业金融终端风格、通达信 .day 导入、回测可视化、盘中预警 MVP、图表细节增强、回测 Bug 修复和多 Agent 安全审计硬化。

下一阶段禁止继续盲目堆功能。不要再优先增加指标数量、策略模板数量、图表炫技功能或复杂量化概念。

下一阶段目标是：

把当前产品从“功能很多的本地量化软件原型”推进为“可被真实用户下载、安装、理解、信任、试用、付费，并且合规风险可控的可卖 Alpha”。

本阶段的核心判断标准：

1. 用户敢不敢安装；
2. 用户装完会不会用；
3. 用户是否理解它不是荐股软件；
4. 产品是否避免直接给出具体股票买卖建议；
5. 是否有清晰风险提示；
6. 是否有可执行的首批用户测试路径；
7. 是否有真实用户反馈闭环；
8. 是否能在无服务器或极低服务依赖下完成试用、交付和授权；
9. 是否能在中国大陆平台发布时避开虚假宣传、荐股、承诺收益、非法投顾等风险；
10. 是否能形成第一版真实销售材料。

一、最高优先级：合规边界重构
====================

  prohibited-copywriting.md

重点任务：

1. 重新审查产品定位，禁止定位为：
   - 荐股软件；
   - 股票推荐软件；
   - 买卖点提示神器；
   - 稳赚工具；
   - 牛股挖掘器；
   - 暴涨预警器；
   - 收益提升工具；
   - 投顾替代品。

2. 允许定位为：
   - 本地离线行情数据分析工具；
   - 本地技术指标学习与复盘工具；
   - 本地回测与历史验证工具；
   - 投资者自我研究辅助软件；
   - 交易复盘与风险教育工具；
   - 非实时、非荐股、非投顾的桌面分析工作站。

3. 全产品文案必须加入边界：
   - 本软件不提供任何证券投资咨询服务；
   - 不构成任何买入、卖出、持有建议；
   - 所有指标、回测、扫描、预警仅用于历史数据分析与用户自我研究；
   - 历史回测不代表未来收益；
   - 用户需独立判断并自行承担投资风险。

4. 所有可能高风险功能重新命名：
   - “智能选股”改为“条件扫描”；
   - “买入信号”改为“条件触发标记”；
   - “卖出信号”改为“退出条件标记”；
   - “牛股评分”改为“技术条件匹配度”；
   - “收益预测”禁止出现；
   - “目标价预测”禁止出现；
   - “推荐股票”禁止出现；
   - “稳赚”“高胜率”“暴涨”“翻倍”等营销词全部禁止。

5. 对所有功能做风险分级：
   - L1：低风险，纯数据展示；
   - L2：中风险，历史统计/回测；
   - L3：高风险，条件扫描/预警/策略模板；
   - L4：禁止，具体个股推荐、预测目标价、承诺收益、代客决策。

6. 对 L3 功能增加强制免责声明和 UI 提示。
7. 删除或隐藏所有 L4 功能。
8. 检查 README、官网、Landing、软件内文案、商品详情页、销售话术、试用页、升级页，全部统一合规表述。

验收标准：

- securities-compliance-audit.md 必须逐条列出风险功能、风险原因、修改方式；
- prohibited-copywriting.md 必须列出至少 100 个禁止词/禁止表达；
- allowed-copywriting.md 必须给出可用替代表达；
- risk-disclaimer.md 必须能直接放进软件启动页、报告导出页、官网页、商品页；
- feature-risk-classification.md 必须覆盖所有已有功能；
- 所有面向用户的文案不得出现“荐股、推荐买入、保证收益、稳赚、暴涨、翻倍、目标价、内幕、主力提前知道”等表达。

====================
三、v0.12 功能只做“体验闭环”，不做功能膨胀
====================

v0.12 允许开发以下功能，但必须服务于真实可用性。

优先级 P0：

1. 后台预警 + Windows 原生通知；
2. 系统托盘；
3. 多周期 K 线；
4. 数据备份/恢复；
5. 首次使用引导；
6. 样例数据一键加载；
7. 错误状态与空状态优化。

注意：

后台预警、Windows 通知、系统托盘必须使用 Tauri 2 官方能力实现。不得写临时 hack。不得引入额外付费服务。不得做联网推送。

P0 细化：

A. 后台预警

必须支持：

- 应用最小化到托盘后继续检查本地数据；
- 用户可启用/禁用后台扫描；
- 用户可设置扫描频率；
- 触发记录写入本地 SQLite；
- 所有触发记录可查看、清除、导出；
- 不得默认开机自启，除非用户明确开启；
- 不得偷偷联网。

B. Windows 通知

必须支持：

- 价格条件触发通知；
- 均线条件触发通知；
- 成交量条件触发通知；
- 通知点击后跳转对应股票图表；
- 通知失败时有软件内 fallback 提示；
- 通知权限不足时提示用户如何开启。

C. 系统托盘

必须支持：

- 显示/隐藏主窗口；
- 暂停/恢复预警扫描；
- 查看最近 5 条预警；
- 退出应用；
- 明确展示后台状态。

D. 多周期 K 线

必须支持：

- 日线；
- 周线；
- 月线；
- 60 分钟；
- 30 分钟；
- 15 分钟；
- 5 分钟；
- 周线/月线通过日线 resample；
- 分钟周期使用 minute_prices 表；
- 2×2 模式下每个画布可独立周期；
- 切换周期不丢失当前股票和画线状态。

E. 数据备份/恢复

必须支持：

- 导出 SQLite 数据库到用户选择目录；
- 恢复前强制备份当前数据库；
- 显示备份文件大小、时间、版本；
- 恢复失败不得破坏现有数据库；
- 每 7 天提醒备份一次；
- 用户可关闭提醒。

F. 首次使用引导

必须支持：

- 选择“加载演示数据”；
- 选择“导入通达信数据”；
- 选择“只看教程”；
- 3 步内让用户看到第一张 K 线图；
- 5 分钟内让用户跑通一次示例回测；
- 不允许新用户打开后面对空白界面。

验收标准：

- 每个 P0 功能必须有测试；
- 每个 P0 功能必须有失败状态；
- 每个 P0 功能必须有用户文档；
- 每个 P0 功能必须有截图或录屏脚本；
- 不完成 P0，不允许进入 P1。
