# 防盗版边界说明

> 版本 v1.0 | 2026-05-24

---

## 一、定位

本文**诚实说明** QuantVault 当前的防盗版措施和边界。

我们不会吹嘘"无法破解"——任何客户端软件理论上都可以被逆向。我们的目标是阻止普通用户"一码多人用"和降低盗版经济激励，不追求绝对防破解。

---

## 二、当前防护措施

### 第一层：Rust 原生二进制

```
Cargo.toml 配置：
[profile.release]
opt-level = "z"       # 最小体积优化
lto = true             # 链接时优化
codegen-units = 1      # 单一编译单元
strip = true           # 剥离调试符号
panic = "abort"        # 不暴露 backtrace
```

**效果**：逆向难度远高于解释型语言（Python/JS/Java），需要专业逆向能力。

### 第二层：JS 混淆

前端代码通过 bundler/minifier 处理（Terser/UglifyJS），变量名缩短、代码压缩。

**效果**：提升阅读门槛，但对有经验者效果有限。

### 第三层：WASM 加密嵌入

核心计算代码编译为 WASM 字节码并加密嵌入，运行时解密加载。

**效果**：WASM 字节码本身已是不易阅读的二进制格式，加密增加一道障碍。

### 第四层：RSA-4096 离线授权

激活码通过 RSA-4096 签名验证，公钥嵌入二进制中。没有私钥无法生成有效激活码。

**效果**：
- 阻止伪造激活码（RSA-4096 短期内无法暴力破解）
- 私钥在开发者处离线保存，不联网不放服务器，泄露风险低

### 第五层：机器指纹绑定

激活码绑定机器硬件指纹（CPU + 主板 + BIOS UUID），一码一机。

**效果**：阻止普通用户分享激活码。

## 三、当前不做的（诚实的局限）

### 3.1 不做的原因

| 措施 | 不做的原因 |
|------|-----------|
| 联网验证 | 违背"A股离线工具"核心卖点 |
| 远程吊销 | 同上，需要服务器 |
| 虚拟机检测 | 误伤合法用户（开发/测试环境） |
| 代码虚拟化（VMProtect/Themida） | 价格高（¥3000+），可能引发杀软误报，用户体验差 |
| 驱动级保护 | 过度，可能不稳定，用户反感 |
| 绑定 MAC/IP/硬盘序列号 | MAC/IP 可轻易修改，用户换硬盘不应影响授权 |
| 不定时网络心跳 | 违背离线承诺 |
| 限时强制更新 | 不尊重用户 |

### 3.2 当前可被攻击的路径

诚实承认当前措施下：

1. **专业逆向者**可通过调试/反汇编绕过授权检查（所有客户端软件都面临的问题）
2. **WASM 解密密钥**存在于调用代码中，理论上可提取
3. **机器指纹**可被 hook/spoof
4. **公钥可被替换**（需要修改二进制中的公钥常量，用自己生成的密钥对替换）

### 3.3 为什么现阶段不做更多

1. **目标用户画像**：中年A股投资者，非专业黑客。会破解 RSA 的人不太可能是我们的付费用户。
2. **价格锚点**：¥199 的定价低于聘请专业逆向者的成本。破解的经济激励几乎为零。
3. **成本收益**：花 ¥10,000 买保护 vs. 可能的盗版损失 ¥2,000，不划算。
4. **用户体验**：每增加一层保护（驱动、虚拟机检测、网络心跳），就可能增加一次杀软误报或用户投诉。

## 四、如果发现明显盗版

### 应对策略

1. **搜索监控**：定期搜索淘宝/闲鱼/拼多多有无未经授权的销售
2. **下架投诉**：发现侵权商品，通过平台投诉下架
3. **水印追踪**（可选）：激活码中嵌入用户可识别信息，如泄露可溯源
4. **不再续发**：如发现某用户大量转售激活码，停止其后续服务（换绑等）
5. **版本更新**：大版本升级时更新保护逻辑，旧版不再维护

### 不采取的措施

- 不做 DMCA/律师函（成本高，震慑力对个人卖家有限）
- 不在软件中埋"销毁数据"后门（违法 + 伤害合法用户）
- 不写恶意代码（违法 + 道德问题）

## 五、防盗版与授权体验的平衡

### 原则

```
用户体验 > 防盗版效果
```

| 场景 | 选择 |
|------|------|
| 防盗版措施可能让合法用户无法激活 | 不采用该措施 |
| 防盗版措施可能导致杀软误报 | 不采用该措施 |
| 防盗版措施需要用户联网 | 不采用该措施 |
| 防盗版措施需要驱动级权限 | 不采用该措施 |
| 防盗版措施导致启动变慢 > 500ms | 不采用该措施 |

### 核心逻辑

每增加 ¥1000 的盗版损失，不值得花 ¥5000 购买保护。

¥199 × 50 份 = ¥10,000 收入。如果因为某个保护措施导致 5 个合法用户无法激活而退款/差评，就损失了 ¥995。更不用说差评的长远影响。

## 六、未来可考虑的改进

当用户量和收入达到一定规模后（如月销量 > 100 份），可以考虑：

1. **购买代码签名证书**（约 ¥2,000-3,500/年）— 主要为了安装信任
2. **轻度反调试**（检测 `IsDebuggerPresent`，非驱动级）— 增加逆向门槛
3. **WASM 运行时完整性校验** — 检测 WASM 是否被替换
4. **激活码使用模式分析** — 离线收集异常使用模式（需要用户同意数据收集）

## 七、总结

| 维度 | 当前状态 |
|------|---------|
| 防伪造激活码 | ✅ RSA-4096 签名，可靠 |
| 防一码多用 | ✅ 机器指纹绑定，有效 |
| 防专业逆向 | ⚠️ Rust 二进制 + WASM 加密，有门槛但不绝对 |
| 防 JS 修改 | ⚠️ 混淆 + 压缩，基本门槛 |
| 防虚拟机克隆 | ❌ 未防护（不加，避免误伤） |
| 用户激活体验 | ✅ 离线，无需联网，快速 |
| 误杀率 | ✅ 不引入第三方保护壳，几乎零误杀 |

---

> **诚实声明**：QuantVault 的离线单机授权模型，在当前 ¥199 定价下，防盗版的正确策略是做好基本防护 + 提供良好体验 + 公平定价，而不是追求技术上的绝对防破解。让用户在正向体验中愿意付费，比用技术手段堵住所有可能漏洞更有效。
