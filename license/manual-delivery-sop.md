# 手动收款→发码→售后 SOP

> 版本 v1.0 | 2026-05-24

---

## 一、整体流程概览

```
用户咨询 → 引导试用 → 试用满意 → 确认购买 → 
收款 → 获取机器码 → 生成激活码 → 发送激活码 → 
确认激活 → 记录存档 → 后续售后
```

## 六、阶段五：生成激活码

### 操作步骤

```bash
# 1. 确认机器码格式
# 用户提供：ABCD-1234-EF56-7890

# 2. 运行生成工具
cd crates/license-tool
cargo run -- --private-key ~/.quantvault/license_private.pem generate \
    --machine-id "ABCD-1234-EF56-7890"

# 3. 复制输出的激活码
# QV-K7X2M9NP-R3WQ8V5T-Y1B4D6F8-H2J9L0N3
```

### 异常处理

如果生成失败，见 `license-failure-cases.md` 开发者侧问题。

```

```

| 字段 | 必填 | 说明 |

| 日期 | ✅ | 交易日期 |

| 平台 | ✅ | 微信/支付宝 |
| 金额 | ✅ | 标准价 ¥199 |

| 激活码 | ✅ | 生成的激活码 |

| 备注 | — | 特殊说明 |

## 十一、退款处理

### 如用户申请退款

1. 确认是否属于退款政策中的可退情形
2. 如属于 → 执行退款 + 注销激活码
3. 如不属于 → 说明原因（引用退款政策）

见 `sales/refund-policy.md` 完整内容。

### 退款操作

```
如审核通过：
1. 原路退款
2. 备注"QuantVault 退款 - [机器码]"
3. 通知用户："退款已处理，激活码已注销，感谢理解"
4. 在销售记录中标记"已退款"
```
