# 回测基准对标案例

> 版本 v1.0 | 2026-05-24 | 10 个手算可验证的基准案例

每个案例：给定已知数据 + 已知策略 + 手工可计算的预期结果 → 与引擎输出对比。

---

## 基准 1：单笔买入持有

**数据**：5 根日线，全涨
```
Bar Open  High  Low   Close Volume
0   10.00 10.50 9.80  10.20 10000
1   10.20 10.80 10.10 10.60 10000
2   10.60 10.90 10.40 10.80 10000
3   10.80 11.20 10.70 11.00 10000
4   11.00 11.50 10.90 11.30 10000
```

**策略**：bar 0 买入，bar 4 卖出
**手续费**：commission=0.0003, stamp_tax=0.001, slippage=0
**初始资金**：100,000
**仓位**：100%

**手算**：
```
Buy: 100000 × (1 - 0.0003) / 10.20 = 9797.06 股（取整）
实际买入成本 = 9797 × 10.20 × (1 + 0.0003) = 99,958.98
剩余现金 = 100,000 - 99,958.98 = 41.02

Sell: 9797 × 11.30 × (1 - 0.0003 - 0.001) = 9797 × 11.30 × 0.9987
    = 9797 × 11.2853 = 110,542. x
约 110,551.9
```

**引擎输出对比**：⬜ 待运行

## 基准 3：连续亏损交易

**数据**：
```
Bar Close (交替跌)
0   10.00
1    9.50
2    9.00
3    8.50
4    8.00
```

**策略**：bar 0 买，bar 2 卖；bar 2 买，bar 4 卖
**手续费**：commission=0, stamp_tax=0, slippage=0（便于手算）

**手算**：
```
Trade 1: Buy 10.00 → Sell 9.00, 亏损 = (9-10)/10 = -10%
Trade 2: Buy 9.00 → Sell 8.00, 亏损 = (8-9)/9 = -11.1%
```

**引擎输出对比**：⬜ 待运行

## 基准 5：最大回撤计算

**数据**：
```
equity_curve: [100000, 102000, 98000, 95000, 97000, 103000, 108000]
```

**手算**：
```
Peak: 100000 → Trough: 95000, DD = -5.00%
Peak: 108000 → (后续无更低价), DD = 0%
Max DD = -5.00%
Max DD Duration = 从 102000 到 95000 = 2 bar
```

## 基准 6：Sharpe Ratio
