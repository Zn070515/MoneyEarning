## 二、基准指标

### P01. 启动时间

**定义**：从双击 .exe 到主窗口渲染完成（首次启动不含数据库迁移）。

| 场景 | 基线 | 警戒线 |
|------|:---:|:---:|
| 冷启动（不含演示数据） | < 2s | < 4s |
| 冷启动（含演示数据） | < 3s | < 5s |
| 热启动（二次启动） | < 1.5s | < 3s |
| 首次启动（含数据库初始化） | < 5s | < 8s |

### P02. K 线渲染

**定义**：从数据加载完成到图表可交互的时间。

| K 线数量 | 指标数 | 基线 | 警戒线 |
|:---:|:---:|:---:|:---:|
| 250 | 0 | < 200ms | < 500ms |
| 250 | 3 (MACD+RSI+BOLL) | < 350ms | < 700ms |
| 1000 | 0 | < 400ms | < 800ms |
| 1000 | 3 | < 600ms | < 1.2s |
| 5000 | 0 | < 1.5s | < 3s |
| 5000 | 3 | < 2.5s | < 5s |

### P03. 缩放/平移帧率

**定义**：K 线图缩放平移时的渲染帧率。

| 操作 | 基线 | 警戒线 |
|------|:---:|:---:|
| 滚轮缩放（单图） | ≥ 30 FPS | ≥ 15 FPS |
| 拖拽平移（单图） | ≥ 40 FPS | ≥ 20 FPS |
| 2×2 网格缩放 | ≥ 20 FPS | ≥ 10 FPS |

### P04. 策略回测

**定义**：从点击"运行回测"到结果展示的时间。

| K 线数量 | 策略复杂度 | 基线 | 警戒线 |
|:---:|------|:---:|:---:|
| 250 | 简单（MA Cross） | < 500ms | < 1s |
| 250 | 复杂（多指标组合） | < 2s | < 4s |
| 1000 | 简单 | < 2s | < 4s |
| 1000 | 复杂 | < 5s | < 10s |
| 5000 | 简单 | < 10s | < 20s |
| 5000 | 复杂 | < 20s | < 40s |

### P05. 数据导入

**定义**：从点击"开始导入"到导入完成的时间。

| 数据量 | 格式 | 基线 | 警戒线 |
|:---:|------|:---:|:---:|
| 100 只股票日线 | 通达信 .day | < 30s | < 1min |
| 500 只股票日线 | 通达信 .day | < 2min | < 5min |
| 1000 只股票日线 | 通达信 .day | < 4min | < 10min |
| 1000 行 | CSV | < 5s | < 15s |
| 50000 行 | CSV | < 30s | < 1min |

### P06. 图标切换/页面导航

| 操作 | 基线 | 警戒线 |
|------|:---:|:---:|
| 切换股票（日线 250 根） | < 300ms | < 800ms |
| 切换周期（日→周） | < 500ms | < 1s |
| 页面间导航 | < 200ms | < 500ms |
| 2×2↔单图切换 | < 500ms | < 1s |

### P07. 内存占用

**定义**：任务管理器→进程→内存（专用工作集）。

| 状态 | 基线 | 警戒线 |
|------|:---:|:---:|
| 空闲（无数据） | < 150 MB | < 250 MB |
| 演示数据加载 | < 250 MB | < 400 MB |
| 5000 只股票日线数据 | < 500 MB | < 800 MB |
| 2×2 网格模式 | < 400 MB | < 600 MB |
| 后台运行 30 分钟 | 增长 < 50 MB | 增长 < 200 MB |
| 后台运行 4 小时 | 增长 < 100 MB | 增长 < 400 MB |

### P08. 安装包大小

| 文件 | 基线 | 警戒线 |
|------|:---:|:---:|
| .exe 安装包 | < 40 MB | < 60 MB |
| .msi 安装包 | < 40 MB | < 60 MB |
| 安装后磁盘占用 | < 150 MB | < 250 MB |

## 三、测量方法

### 启动时间
```
Windows Performance Recorder 或 PowerShell Measure-Command
测量：进程创建到窗口显示
```

### 图表渲染
```
Rust: std::time::Instant （前后插入计时 log）
前端: performance.now() / performance.mark()
```

### 内存
```
任务管理器 → 详细信息 → 专用工作集
或 PowerShell: Get-Process QuantVault | Select-Object WS
```

### 安装包大小
```
dir /B /S *.exe *.msi
文件属性 → 大小
```

## 五、回归检测

如果某版本较上一版本：
- 启动时间增长 > 50% → 调查
- 回测时间增长 > 100% → 调查
- 内存占用增长 > 50% → 调查
- 安装包增大 > 20 MB → 调查

---

> **性能下降也是 Bug。不要在不知不觉中让软件越来越慢。**
